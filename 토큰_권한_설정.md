# 🔐 Classic Token 권한 설정 가이드

## ✅ 체크해야 할 권한

### 필수 권한

**`repo`** - 이것만 체크하면 됩니다!

`repo` 권한을 체크하면 다음 하위 권한이 자동으로 포함됩니다:
- ✅ `repo:status` - 저장소 상태 접근
- ✅ `repo_deployment` - 배포 접근
- ✅ `public_repo` - Public 저장소 접근
- ✅ `repo:invite` - 초대 관리
- ✅ `security_events` - 보안 이벤트

### 선택 권한 (필요 없음)

다음은 체크하지 않아도 됩니다:
- ❌ `workflow` - GitHub Actions (선택사항)
- ❌ `write:packages` - 패키지 업로드 (필요 없음)
- ❌ `delete_repo` - 저장소 삭제 (필요 없음)
- ❌ 기타 모든 권한 (필요 없음)

## 📋 설정 방법

1. **토큰 생성 페이지**
   - https://github.com/settings/tokens 접속
   - "Generate new token (classic)" 클릭

2. **Note 입력**
   - `studyground-deploy` (아무 이름)

3. **Expiration 선택**
   - `90 days` 또는 원하는 기간

4. **Select scopes (권한 선택)**
   - ✅ **`repo`** 체크 (이것만!)
   - 나머지는 모두 체크 해제

5. **"Generate token" 클릭**

6. **토큰 복사**
   - `ghp_`로 시작하는 토큰 복사
   - 한 번만 보여주므로 즉시 복사!

## 🎯 요약

**체크할 것:**
- ✅ `repo` (하나만!)

**체크하지 않을 것:**
- ❌ 나머지 모든 권한
