## 🚀 지금 바로 시작하기

### 1단계: Google Sheets 설정 (5분)
[GOOGLE_SHEETS_SETUP.md](./GOOGLE_SHEETS_SETUP.md) 파일을 참고하여:
1. Google Cloud Console에서 프로젝트 생성
2. Google Sheets API 활성화
3. 서비스 계정 생성 및 JSON 키 다운로드
4. Google Sheets 스프레드시트 생성
5. 서비스 계정에 편집 권한 부여

### 2단계: 환경 변수 설정 (2분)
프로젝트 루트에 `.env` 파일 생성:

```env
GOOGLE_SHEETS_ID=스프레드시트_ID_여기에_입력
GOOGLE_SERVICE_ACCOUNT={"type":"service_account",...JSON_전체_내용...}
JWT_SECRET=openssl_rand_-hex_32_로_생성한_값
PORT=5001
NODE_ENV=development
```

**JWT_SECRET 생성:**
```bash
openssl rand -hex 32
```

### 3단계: 실행 (1분)
```bash
# 의존성 설치
npm install

# 개발 모드 실행
npm run dev

# 또는 프로덕션 모드
npm run build
npm start
```

### 4단계: 접속
- **키오스크**: http://localhost:5001/kiosk
- **관리자**: http://localhost:5001/admin
  - 아이디: `admin`
  - 비밀번호: `admin1234`
- **공용 대시보드**: http://localhost:5001/public

## ⚠️ 중요 사항

1. **보안**: 프로덕션 환경에서는 반드시:
   - 관리자 비밀번호 변경
   - 강력한 JWT_SECRET 사용
   - HTTPS 사용
   - `.env` 파일 공개 금지

2. **Google Sheets 권한**: 서비스 계정에 스프레드시트 편집 권한이 있어야 합니다.

3. **포트 충돌**: 다른 애플리케이션이 5001 포트를 사용 중이면 `.env`에서 포트 변경

## 🆘 문제 해결

### Google Sheets 연결 오류
- 서비스 계정 JSON 파일 확인
- 스프레드시트 ID 확인
- 서비스 계정 권한 확인

### 빌드 오류
- Node.js 버전 확인 (v16 이상)
- `rm -rf node_modules && npm install` 재설치

### 접속 불가
- 서버가 실행 중인지 확인
- 포트 번호 확인
- 방화벽 설정 확인
