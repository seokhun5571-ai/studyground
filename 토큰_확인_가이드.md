# 🔐 토큰 생성 및 확인 가이드

## ⚠️ 문제 해결

토큰이 제대로 작동하지 않는다면 다음을 확인하세요:

## 1. 토큰 생성 확인

### 올바른 토큰 생성 방법

1. **정확한 페이지 접속**
   - https://github.com/settings/tokens
   - 또는: GitHub → 우측 상단 프로필 → Settings → 좌측 메뉴 하단 "Developer settings" → "Personal access tokens" → "Tokens (classic)"

2. **토큰 생성**
   - "Generate new token (classic)" 클릭 (⚠️ classic 버전!)
   - Note: `studyground-deploy`
   - Expiration: `90 days` 또는 `No expiration` (테스트용)
   - **Scopes (권한)**: 
     - ✅ `repo` 체크 (전체 repo 권한)
     - ✅ `workflow` 체크 (GitHub Actions용, 선택사항)
   - "Generate token" 클릭

3. **토큰 복사**
   - 생성된 토큰은 **한 번만** 보여줍니다!
   - 토큰은 `ghp_` 로 시작하는 긴 문자열입니다
   - 예: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
   - **즉시 복사**하세요!

## 2. 토큰 형식 확인

올바른 토큰 형식:
- ✅ `ghp_` 로 시작
- ✅ 약 40자 이상의 긴 문자열
- ✅ 공백 없음

잘못된 예:
- ❌ 비밀번호
- ❌ 사용자명
- ❌ 짧은 문자열

## 3. 권한 확인

토큰에 다음 권한이 있어야 합니다:
- ✅ `repo` (필수)
  - `repo:status`
  - `repo_deployment`
  - `public_repo`
  - `repo:invite`
  - `security_events`

## 5. 문제 해결

### "Invalid username or token" 오류

**원인:**
- 토큰이 올바르지 않음
- 토큰 권한 부족
- 토큰 만료

**해결:**
1. 새 토큰 생성
2. `repo` 권한 확인
3. 토큰 전체를 정확히 복사

### "Authentication failed" 오류

**원인:**
- 사용자명 오류
- 토큰 형식 오류

**해결:**
1. 사용자명 확인: `seokhun5571-ai` (lonnie 아님!)
2. 토큰이 `ghp_` 로 시작하는지 확인
3. 토큰에 공백이 없는지 확인

### 토큰이 작동하지 않음

**체크리스트:**
- [ ] 토큰이 `ghp_` 로 시작하는가?
- [ ] 토큰이 40자 이상인가?
- [ ] `repo` 권한이 체크되어 있는가?
- [ ] 토큰이 만료되지 않았는가?
- [ ] 토큰을 정확히 복사했는가? (앞뒤 공백 없음)

## 6. 새 토큰 생성

기존 토큰이 작동하지 않으면:

1. https://github.com/settings/tokens 접속
2. 기존 토큰 삭제 (선택사항)
3. "Generate new token (classic)" 클릭
4. 새 토큰 생성 및 복사
5. 다시 푸시 시도
